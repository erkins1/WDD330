Testing and Debugging

Exceptions - returns a value to describe the problem

Stack Traces - returned to identify where the error happened

Warnings - Potential issue that does not crash program

Linting Tools - Help to prevent sloppy programming

Feature Detection - Make sure methods or libraries exist before using them

Alerts can stop the program from running until 'Okay' is clicked. This is useful to see code in action.

Console.log is also a great way to debug by outputting variables

debugger; pauses the code until you start it again

Error object also has some resources

try catch blocks help the code to run even when an error is thrown

Test-driven development is the concept of writing tests to make sure code works before writing code.
