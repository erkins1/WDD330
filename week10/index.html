Client-side form validation

Form validation is done to make sure that inputs are correct

Client side is done so the user can fix it

Server side id done to protect the

Types of validation include:

required fields, correct formats, password reqirements

Built-in form validation includes

required

minlength/maxlength

min/max (for numerical data)

type (email, number, etc.)

pattern (uses a regular expression)

Elements can match a :valid or :invalid CSS class

The Constraint Validation API can be used to validate with javascript

The API includes DOM elements like the HTMLButtonElement

Button, FieldSet, Input, Output, Select, TextArea

There's a wide range of properties that help determine validity

Custom error messages are possible, but cannot be changed with CSS

Using Fetch

3 Differences between fetch() and jQuery.ajax():

Fetch does not reject an HTTP error status

Fetch can(?) receive cross-site cookies

Fetch won't send cookies

Fetch has 2 parameters, a url and a data object

The data object can be:

credentials

json data

methods (GET/POST)

files

A successful fetch response includes an response.ok if it was successful

Headers() object creates headers that can be passed along in the fetch

Response objects are returned when the fetch promise is resolved

requests and resposes can include body data
